  DESCRIPTION  (min 280 chars)
════════════════════════════════════════════════════════════════════

ClawForger is an open agent framework where every agent is a transferable, evolving, monetizable economic entity — not just a process.

Each ClawForger agent is minted as an ERC-7857 iNFT on 0G Galileo, with its system prompt, learned skills, and memory stored encrypted on 0G Storage. When an agent encounters a task its existing skills can't solve, it generates new tool code, sandbox-tests it against the task's success criteria, and on success publishes the artifact to an on-chain skill registry. The iNFT metadata updates dynamically via ERC-7857 + ERC-4906.

Other agents discover that skill in the registry and pay to use it: each skill is gated by an HTTP 402 paywall, with USDC micropayments routed through the x402 protocol. Royalties auto-split 95/5 to the iNFT owner / protocol via an on-chain RoyaltyVault per agent.

Every onchain action — minting, evolution, skill registration, royalty settlement — flows through KeeperHub's MCP execution layer for guaranteed delivery, gas optimization, retry safety, and audit trail. No package in the framework signs a raw transaction directly; every action compiles to a KeeperHub workflow descriptor.

The result is a working agent economy: a Researcher agent evolves a fetch.arxiv skill, publishes it to the on-chain SkillRegistry at 0.05 mUSDC, a Writer agent discovers it via the Studio marketplace, pays via x402, KeeperHub settles, the Researcher's owner receives 0.0475 mUSDC. Six sponsor primitives, one continuous flow, demoable in 90 seconds.

  HOW IT'S MADE  (min 280 chars)
════════════════════════════════════════════════════════════════════

ClawForger is a Bun + TypeScript monorepo with seven workspace packages and four Solidity contracts. Smart contracts (ClawforgerINFT, SkillRegistry, RoyaltyVault, MUSDC) are deployed via Foundry to 0G Galileo with viaIR optimization enabled — needed because our tokenURI generates a base64-encoded JSON metadata blob with an inline SVG NFT card per ERC-721 standard, which exceeded the EVM stack depth without IR.

The on-chain NFT card is fully self-contained: linear-gradient navy background, purple aurora (0G Purple 1, #B75FFF), inset rounded frame, four corner-bracket marks, and the geometric brand mark (claw + anvil + ember + eye) embedded as raw SVG paths. No IPFS, no gateway — render it on any ERC-721-aware explorer and the iNFT shows up looking like a passport stamp.

Off-chain packages: @clawforger/core (runtime + canonical types), @clawforger/inft-identity (ERC-7857 SDK with our own resilient tx-receipt poller, since 0G's public RPC has eventual-consistency hiccups that throw viem's default waitForTransactionReceipt), @clawforger/memory-0g (KV + Log adapter with client-side AES-256-GCM encryption, key derived from iNFT owner's signature via HKDF), @clawforger/skill-forge (the self-evolution loop with Bun-worker sandbox, 10s timeout, schema validation), @clawforger/keeperhub-execute (the KeeperHub MCP client with viem fallback for KH-shaped retry semantics if their REST API rejects), @clawforger/x402-facilitator (the first 0G x402 facilitator we know of — verifies EIP-712 typed-data payment authorizations against on-chain mUSDC allowance), and @clawforger/x402-skill-market (paywalled HTTP 402 server with auto-sync from on-chain SkillRegistry).

The Studio is Vite 6 + React 19 + wagmi 2 + RainbowKit on 0G Galileo (single chain, no second chain). It auto-loads addresses.json, queries on-chain events (AgentMinted, AgentEvolved, SkillPublished) for live data, and uses the brand mark + 0G purple accent everywhere.

Hackiest parts worth mentioning: (1) Our KeeperHub bridge attempts the documented POST /workflows/create endpoint, which returns 404 — a real bug we shipped in FEEDBACK.md. We compile the workflow JSON regardless and fall through to viem with KH-shaped retry. (2) Solidity string literals can't hold UTF-8 multi-byte chars, so the middle-dot in our chain stamp ("ERC-7857 · 0G GALILEO") is embedded as the raw bytes \xc2\xb7. (3) Examples wire an onSkillPublish hook that auto-registers evolved skills on the on-chain SkillRegistry — so the Studio marketplace updates instantly without a manual step. (4) The x402 facilitator we ship is the first one we're aware of for 0G — Coinbase's official one is Base-only.

Partner tech in load-bearing roles: 0G Chain (ERC-7857 iNFT, SkillRegistry, RoyaltyVault all live there), 0G Storage (encrypted skill artifacts + memory log), 0G Compute (verified TEE inference via the @0glabs/0g-serving-broker, with MockInference fallback for offline demos), KeeperHub (MCP-based execution layer for every onchain action), and x402 (HTTP 402 paywall standard for skill payments). Each integration is wired into the demo path — none are decorative.
